# Описание класса: ViewResultActivity

## 1. Общая информация
*   **Имя класса:** `ViewResultActivity`
*   **Тип:** Activity (Экран)
*   **Назначение:** Этот экран показывает финальный результат сложения двух волн. Он отображает большую анимированную графику на основе параметров, выбранных в `JunctionActivity`.
*   **Взаимодействие:** Получает данные от `JunctionActivity` и использует `GraphView` для отрисовки.

## 2. Переменные (Поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `graphView` | `GraphView` | Основная область рисования, где происходит анимация. | `onCreate` |

## 3. Методы класса

### Имя метода: `onCreate`
*   **Логика:**
    1.  Устанавливает макет `activity_view_result.xml`.
    2.  Включает кнопку "Назад" в верхней панели.
    3.  Извлекает 6 параметров волн из объекта `Intent`.
    4.  Вызывает `graphView.setParameters()`, чтобы запустить анимацию.

## 4. Жизненный цикл
*   **`onCreate()`:** Инициализирует финальный график при открытии экрана.

## 5. Взаимодействие с интерфейсом (UI)
*   **`GraphView`:** Занимает почти весь экран для показа результата.

## 6. Взаимодействие с другими компонентами
*   **`Intent`:** Принимает "пакет" данных, отправленный из `JunctionActivity`.

## 7. Общая логика
Этот класс просто берет "рецепт" (параметры), который он получил, и передает его "повару" (`GraphView`), чтобы тот приготовил финальное "блюдо" (анимированный график).

## 8. Объяснение простыми словами
**Аналогия:** Представьте, что вы сконструировали модель машины в каталоге (`JunctionActivity`). `ViewResultActivity` — это **Выставочный зал**, где вы видите, как эта машина едет.
