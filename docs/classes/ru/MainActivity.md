# Описание класса: MainActivity

## 1. Общая информация
*   **Имя класса:** `MainActivity`
*   **Тип:** Activity (Активность/Экран)
*   **Назначение:** Этот класс является "входной дверью" приложения. Он отображает заставку (Splash Screen) — приветственный экран с коротким видео, прежде чем перевести пользователя к настройкам.
*   **Взаимодействие:** Запускает жизненный цикл приложения и переходит к `JunctionActivity` после задержки или окончания видео.

## 2. Переменные (Поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `SPLASH_DELAY` | `int` (константа) | Длительность ожидания (4000 мс = 4 сек) перед автоматической сменой экрана. | `onCreate` |
| `activityStarted` | `boolean` | "Флаг" (проверка), чтобы не запустить следующий экран дважды (например, если видео кончилось и таймер сработал одновременно). | `startJunctionActivity` |

## 3. Методы класса

### Имя метода: `onCreate`
*   **Тип:** `protected`
*   **Возвращаемое значение:** `void` (ничего)
*   **Параметры:** `Bundle savedInstanceState` (стандартный объект для восстановления состояния экрана).
*   **Логика:**
    1.  Вызывает `super.onCreate` для базовой настройки.
    2.  Включает режим `EdgeToEdge` (контент заходит под системные панели для красоты).
    3.  Устанавливает внешний вид из файла `activity_main.xml`.
    4.  Инициализирует `VideoView` для воспроизведения видео `twosin` из папки ресурсов.
    5.  Настраивает "слушатель": если видео закончилось, вызвать `startJunctionActivity`.
    6.  Запускает таймер (`Handler`), который вызовет `startJunctionActivity` через 4 секунды в любом случае.
*   **Когда вызывается:** Автоматически системой при запуске приложения.

### Имя метода: `startJunctionActivity`
*   **Тип:** `private`
*   **Логика:**
    1.  Проверяет, не был ли экран уже запущен (`activityStarted`).
    2.  Если нет, меняет флаг на `true`.
    3.  Создает `Intent` (намерение) для открытия `JunctionActivity`.
    4.  Запускает новый экран и вызывает `finish()` (закрывает текущий, чтобы нельзя было вернуться к заставке кнопкой "Назад").
*   **Когда вызывается:** По окончании видео или по таймеру.

## 4. Жизненный цикл
*   **`onCreate()`:** Вызывается при старте. Здесь настраивается видео и таймер.
*   **`onDestroy()`:** Происходит при вызове `finish()`, чтобы освободить память.

## 5. Взаимодействие с интерфейсом (UI)
*   **`VideoView`:** Плеер для вступительного видео. Связан через `findViewById(R.id.videoView)`.
*   **`RainbowTextView`:** Текстовое поле с радужным градиентом (кастомный элемент).

## 6. Взаимодействие с другими компонентами
*   **`Intent`:** Используется для перехода от `MainActivity` к `JunctionActivity`.

## 7. Общая логика
При открытии приложения `MainActivity` показывает видео-заставку на несколько секунд, а затем быстро "передает эстафету" классу `JunctionActivity`, закрываясь само.

## 8. Объяснение простыми словами
**Аналогия:** Представьте вход в кинотеатр. `MainActivity` — это заставка кинокомпании (логотип), которую вы видите, пока усаживаетесь. Как только она заканчивается, начинается сам фильм (основное приложение), а заставка исчезает.
