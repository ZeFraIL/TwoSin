# Описание класса: GraphView

## 1. Общая информация
*   **Имя класса:** `GraphView`
*   **Тип:** Custom View (Пользовательский графический элемент)
*   **Назначение:** Это "мозг" и "холст" приложения. Он выполняет сложные математические расчеты синусоид и фигур Лиссажу и рисует их на экране попиксельно.
*   **Взаимодействие:** Используется внутри `JunctionActivity` (для превью) и в `ViewResultActivity` (для финальной анимации).

## 2. Переменные (Поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `paint` | `Paint` | "Кисть" для рисования красной линии графика. | `init`, `onDraw` |
| `axisPaint` | `Paint` | "Карандаш" для рисования черных осей координат (X и Y). | `init`, `onDraw` |
| `animator` | `ValueAnimator` | Таймер, который плавно меняет значение от 0 до 1 в течение 10 секунд для анимации рисования. | `init`, `setParameters` |
| `amp1`, `freq1`... | `float` | Хранят математические параметры волн (амплитуду, частоту и т.д.). | `onDraw`, `setParameters` |

## 3. Методы класса

### Имя метода: `init`
*   **Тип:** `private void`
*   **Логика:** Настраивает цвет и толщину "кистей" (`Paint`) и подготавливает аниматор.
*   **Когда вызывается:** При создании объекта (при запуске экрана).

### Имя метода: `setParameters`
*   **Тип:** `public void`
*   **Параметры:** Тип графики и 6 чисел (параметры волн).
*   **Логика:**
    1.  Переводит "целые" числа с ползунков в математические значения (например, градусы в радианы).
    2.  Сбрасывает таймер анимации (`animator`).
    3.  Запускает рисование заново.
*   **Когда вызывается:** Когда нужно начать новую отрисовку.

### Имя метода: `onDraw`
*   **Тип:** `protected void`
*   **Параметры:** `Canvas canvas` (Виртуальный холст/бумага).
*   **Логика:**
    1.  **Рисование осей:** Рисует вертикальную и горизонтальную линии.
    2.  **Расчет и отрисовка графика:**
        *   Если режим "одна ось": Складывает значения двух синусов `y = sin(x1) + sin(x2)` для каждой точки.
        *   Если режим "перпендикулярно": Рисует фигуру Лиссажу, где одна волна двигает "перо" влево-вправо, а другая — вверх-вниз.
    3.  **Анимация:** Рисует только ту часть графика, которую "разрешил" аниматор (например, если прошло 5 секунд из 10, отрисована будет половина).
*   **Когда вызывается:** Система вызывает его каждый раз, когда нужно обновить картинку на экране.

## 4. Жизненный цикл
*   **`onDraw()`:** Основной метод, вызывается многократно во время анимации.
*   **`onDetachedFromWindow()`:** Вызывается, когда экран закрывается. Здесь мы останавливаем аниматор, чтобы не тратить заряд батареи.

## 5. Взаимодействие с интерфейсом (UI)
Это чисто графический компонент. У него нет кнопок, он просто выполняет команды отрисовки, которые ему дают Activity.

## 6. Взаимодействие с другими компонентами
Получает данные от `JunctionActivity` или `ViewResultActivity` через метод `setParameters`.

## 7. Общая логика
Класс получает числа -> превращает их в координаты с помощью тригонометрии (`Math.sin`) -> соединяет точки линиями на холсте (`Canvas`).

## 8. Объяснение простыми словами
**Аналогия:** Это как **Робот-Художник**. Активности дают ему "Инструкцию" (насколько сильно и быстро двигать рукой), а `GraphView` берет маркер и ведет линию по бумаге, кадр за кадром создавая сложный узор.
