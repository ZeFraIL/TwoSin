# Описание класса: JunctionActivity

## 1. Общая информация
*   **Имя класса:** `JunctionActivity`
*   **Тип:** Activity (Экран)
*   **Назначение:** Это "панель управления" приложения. Здесь пользователь настраивает математические свойства (амплитуду, частоту, фазу) двух разных волн и видит их предпросмотр.
*   **Взаимодействие:** Принимает эстафету от `MainActivity` и передает выбранные параметры в `ViewResultActivity` для финальной отрисовки.

## 2. Переменные (Поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `seekBarAmplitude1` | `SeekBar` | Ползунок для изменения высоты (амплитуды) первой волны. | `setupViews`, `openResultActivity` |
| `textViewFrequency1` | `TextView` | Текст для отображения текущего значения частоты. | `updateTextViews` |
| `graphView1` | `GraphView` | Мини-график для предпросмотра первой волны. | `updateGraph1` |
| `buttonSameAxis` | `Button` | Кнопка для перехода к сложению волн на одной оси. | `setupListeners` |

*(Аналогичные переменные есть для второй волны и других параметров).*

## 3. Методы класса

### Имя метода: `onCreate`
*   **Тип:** `protected void`
*   **Логика:** Устанавливает макет экрана, вызывает вспомогательные методы для связи с UI (`setupViews`) и настройки реакции на действия пользователя (`setupListeners`).
*   **Когда вызывается:** Сразу после закрытия `MainActivity`.

### Имя метода: `setupListeners`
*   **Тип:** `private void`
*   **Логика:** Создает "слушателей" для ползунков. Каждый раз, когда вы двигаете ползунок, код автоматически вызывает `updateGraph()` и `updateTextViews()`. Это создает эффект "живого" изменения графиков.
*   **Когда вызывается:** Один раз при создании экрана.

### Имя метода: `updateGraph1` / `updateGraph2`
*   **Тип:** `private void`
*   **Логика:** Берет текущее положение ползунка (0-100) и передает его в `GraphView`, чтобы тот перерисовал волну с новыми данными.
*   **Когда вызывается:** При каждом движении ползунка.

### Имя метода: `openResultActivity`
*   **Тип:** `private void`
*   **Параметры:** `String type` (режим сложения: на одной оси или перпендикулярно).
*   **Логика:** "Упаковывает" все 6 параметров (амплитуды, частоты, фазы обеих волн) в объект `Intent` (как письмо в конверт) и отправляет их на экран результатов.

## 4. Жизненный цикл
*   **`onCreate()`:** Используется для создания интерфейса и подготовки ползунков к работе.

## 5. Взаимодействие с интерфейсом (UI)
*   **`SeekBar` (Ползунок):** Для ввода данных пользователем.
*   **`TextView`:** Для показа числовых значений.
*   **`Button`:** Для перехода к финальной анимации.
*   **`GraphView`:** Для отображения мини-превью волн.

## 6. Взаимодействие с другими компонентами
*   **`Intent`:** Передает все математические данные в `ViewResultActivity` через функцию `putExtra()`.

## 7. Общая логика
Пользователь двигает ползунки -> Код замечает изменения -> Мини-график и текст обновляются мгновенно -> Пользователь жмет кнопку -> Все данные отправляются на следующий экран для большой анимации.

## 8. Объяснение простыми словами
**Аналогия:** Представьте **Радиоприемник**. `JunctionActivity` — это передняя панель с ручками громкости и настройки частоты. Когда вы крутите ручки, вы видите, как меняются цифры на табло. Когда вы настроились, вы нажимаете кнопку "Играть", чтобы звук пошел на большие колонки.
