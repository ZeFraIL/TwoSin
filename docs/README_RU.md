🎯 Цель проекта
•	Приложение решает задачу визуализации математического сложения двух синусоидальных колебаний.
•	Эта задача важна для студентов и инженеров, изучающих физику, электронику и обработку сигналов (принцип суперпозиции и фигуры Лиссажу).
•	Целевая аудитория: Студенты, преподаватели, любители науки.
________________________________________
📌 Требования к приложению
**Функциональные требования**
•	Отображение заставки с видео (Splash screen).
•	Интерактивная настройка параметров двух волн (Амплитуда, Частота, Фаза) с помощью SeekBar.
•	Предварительный просмотр каждой волны в реальном времени.
•	Два режима визуализации результата: сложение на одной оси и на перпендикулярных осях.
•	Анимированное построение результирующего графика.

**Нефункциональные требования**
•	Производительность: Плавная отрисовка графиков на Canvas.
•	Удобство использования: Простой интерфейс управления на одном экране.
•	Надёжность: Корректное управление ресурсами аниматора и жизненным циклом Activity.
________________________________________
🧠 Общая архитектура
•	**Выбранный подход:**
–	MVC (Model-View-Controller).
•	**Почему выбран именно он:**
–	Обеспечивает четкое разделение между математической логикой (Model), пользовательским графическим элементом (View) и управлением потоками данных (Controller).
•	**Основные компоненты системы:**
–	`MainActivity`: Логика экрана приветствия.
–	`JunctionActivity`: Экран ввода параметров и предпросмотра.
–	`ViewResultActivity`: Экран отображения итоговой анимации.
–	`GraphView`: Ядро математических вычислений и отрисовки.
________________________________________
🧩 UML-диаграмма (обязательно)
[MainActivity] –> [JunctionActivity]
[JunctionActivity] –> [ViewResultActivity]
[ViewResultActivity] –> [GraphView]
________________________________________
**Объяснение:**
- Логика отрисовки инкапсулирована в `GraphView`, что гарантирует идентичность предпросмотра и финального результата.
- Использование Intent extras позволяет легко передавать параметры между экранами.
________________________________________
🧩 Подробное описание классов
📌 **Класс: MainActivity**
**Роль:**
Точка входа и презентация приложения.
**Ответственность:**
Воспроизведение вступительного видео и переход к основному интерфейсу.
**Основные методы:**
- `onCreate()`: Инициализация VideoView.
- `startJunctionActivity()`: Логика перехода по завершении видео или таймеру.

📌 **Класс: JunctionActivity**
**Роль:**
Центр управления конфигурацией.
**Ответственность:**
Сбор данных от пользователя через SeekBars и мгновенное обновление мини-графиков.
**Основные методы:**
- `setupListeners()`: Реактивное обновление UI.
- `updateTextViews()`: Синхронизация текстовых значений параметров.

📌 **Класс: GraphView**
**Роль:**
Движок рендеринга.
**Ответственность:**
Отрисовка осей координат, расчет тригонометрических функций и анимация траектории.
**Основные методы:**
- `setParameters()`: Обновление параметров и перезапуск аниматора.
- `onDraw()`: Поточечная отрисовка кадра.
________________________________________
🔄 Диаграмма работы приложения
1. Приложение запускается с 4-секундным видео-вступлением.
2. Пользователь настраивает свойства двух волн с помощью ползунков.
3. Пользователь выбирает тип сложения (на одной оси или перпендикулярно).
4. Приложение пошагово рисует результирующую траекторию.

🎨 UI/UX анализ
•	Интерфейс спроектирован для максимальной наглядности и мгновенной обратной связи.
•	**Какие принципы использованы:**
–	Простота: Только необходимые элементы управления.
–	Логичность: Последовательный путь от настройки к результату.
–	Доступность: Крупные элементы управления и четкие подписи.
•	**Что можно улучшить:** Добавить возможность сохранения графиков в виде изображений.
________________________________________
⚙️ Работа с потоками
**Используются:**
- `ValueAnimator`: Для плавного обновления кадров анимации.
- `Handler`: Для задержки на экране заставки.
•	**Как предотвращаются:**
–	зависания (ANR): Математические расчеты оптимизированы для выполнения в реальном времени.
–	утечки памяти: Аниматоры останавливаются в `onDetachedFromWindow`.
________________________________________
💾 Работа с данными
•	**Способ:** Передача через Intent Extras.
•	**Почему:** Идеально подходит для передачи небольших наборов примитивных параметров.
________________________________________
🌐 Работа с сетью
•	Приложение работает полностью автономно (Offline).
________________________________________
🔐 Безопасность (базовый уровень)
•	Конфиденциальные данные не используются.
________________________________________
🧪 Тестирование
•	Ручное тестирование математической точности графиков (сравнение с известными фигурами Лиссажу).
•	Проверка жизненного цикла при повороте экрана.
________________________________________
🐞 Обработка ошибок
•	Инициализация параметров по умолчанию для предотвращения сбоев при пустых Intent.
•	Ограничения SeekBar предотвращают ввод некорректных математических значений.
________________________________________
⚡ Производительность
•	Оптимизация: Расчет точек происходит "на лету" во время отрисовки, что минимизирует использование памяти.
________________________________________
🚀 Возможности расширения
•	Поддержка сложения более чем двух волн.
•	Интерактивное изменение параметров перетаскиванием точек на графике.
•	Настройка цветов для каждой волны.
